## Overview

Del Poble Pizzeria uses Sanity schemas to define the structure of all content. Schemas are located in `delpoble-studio/schemas/` and define the fields, validation, and organization of content.

## Schema Files

The main schema files include:

- `frontPage.ts` - Homepage content structure
- `cartaPage.ts` - Menu page with sections and categories
- `dish.ts` - Individual dish/product definition
- `historiaPage.ts` - History page content
- `pizzeriasPage.ts` - Locations page
- `contactPage.ts` - Contact page
- `trabajaConNosotrosPage.ts` - Careers page
- `generalPage.ts` - Dynamic general pages
- `footer.ts` - Footer configuration
- `siteSettings.ts` - Global site settings
- `localized.ts` - Localization helpers

## Carta Page Schema

The menu page (`cartaPage.ts`) uses nested arrays to organize dishes:

### Structure

<ParamField path="sections" type="array" required>
  Array of menu sections (e.g., Pizzas, Entrantes, Bebidas)
  
  <Expandable title="section fields">
    <ParamField path="title" type="localizedString" required>
      Section title (e.g., "PIZZES")
    </ParamField>
    
    <ParamField path="imageTitle" type="localizedString">
      Text displayed with the featured image
    </ParamField>
    
    <ParamField path="featuredImage" type="image">
      Main image for the section with alt text and hotspot
    </ParamField>
    
    <ParamField path="order" type="number" required>
      Display order (integer, min: 0)
    </ParamField>
    
    <ParamField path="categories" type="array">
      Subcategories within the section
      
      <Expandable title="category fields">
        <ParamField path="availability" type="array">
          Where this category is available:
          - `delivery` - Available for delivery
          - `pizzeria` - Available at pizzeria locations
          - `restaurante` - Available at restaurant locations
          - `consultar` - Check at location
        </ParamField>
        
        <ParamField path="name" type="localizedString">
          Category name (e.g., "Clàssic", "Gourmet")
        </ParamField>
        
        <ParamField path="specialBadge" type="image">
          Special badge/icon displayed after category name
        </ParamField>
        
        <ParamField path="order" type="number" required>
          Display order within section
        </ParamField>
        
        <ParamField path="dishes" type="array">
          References to dish documents
        </ParamField>
      </Expandable>
    </ParamField>
  </Expandable>
</ParamField>

### Example Structure

```typescript
// Example: Carta Page structure
{
  sections: [

}
```

## Localized Field Helpers

The `localized.ts` file provides helper functions for creating multilingual fields:

### localizedString

Creates a string field with translations:

```typescript
localizedString('fieldName', 'Field Title', 'Optional description')
```

Generates:
- `es` (Español) - Required
- `en` (English) - Optional
- `val` (Valencià) - Optional

### localizedText

Creates a textarea field with translations:

```typescript
localizedText('description', 'Description', 10) // 10 rows
```

### localizedArray

Creates an array field with localized content:

```typescript
localizedArray('items', 'Items', {
  type: 'object',
  fields: [...]
})
```

### localizedResponsiveText

Creates responsive text with device-specific translations:

```typescript
localizedResponsiveText('highlightedText', 'Highlighted Text', 4)
```

Generates fields for:
- Desktop (required)
- Tablet (optional, ≤1000px)
- Mobile (optional, ≤600px)

Each with full language support (es, en, val).

### localizedResponsiveArray

Same as responsive text, but for arrays of objects.

## Field Validation

Schemas include validation rules:

```typescript
defineField({
  name: 'order',
  type: 'number',
  validation: (Rule) => Rule.required().integer().min(0),
})
```

Common patterns:
- `.required()` - Field must have a value
- `.integer()` - Must be a whole number
- `.min(0)` - Minimum value
- `.max()` - Maximum value
- Custom validators for complex logic

## Image Fields

All image fields support:

```typescript
{
  type: 'image',
  options: {
    hotspot: true, // Enable focal point selection
  },
  fields: [
    {
      name: 'alt',
      type: 'string',
      title: 'Texto alternativo',
    },
  ],
}
```

## Icons

Schemas use React Icons for visual identification:

```typescript
import { MdHome } from 'react-icons/md'

export default defineType({
  name: 'frontPage',
  icon: MdHome,
  // ...
})
```

Available icons:
- `MdHome` - Front Page
- `MdRestaurantMenu` - Carta Page
- `MdRestaurant` - Dish
