## Overview

Del Poble Pizzeria uses Sanity Studio as its headless CMS for managing all website content. The Studio is located in the `delpoble-studio/` directory and provides a user-friendly interface for content editors.

## Configuration

The Sanity Studio is configured via `sanity.config.ts`:

```typescript delpoble-studio/sanity.config.ts
import {defineConfig} from 'sanity'
import {structureTool} from 'sanity/structure'
import {visionTool} from '@sanity/vision'
import {schemaTypes} from './schemas'

export default defineConfig({
  name: 'default',
  title: 'Del Poble Pizzeria',
  
  projectId: 'v8qkjh2b',
  dataset: 'production',
  
  plugins: [
    structureTool({
      structure: (S) => S.list().title('Contenido').items([...])
    }),
    visionTool(),
  ],
  
  schema: {
    types: schemaTypes,
  },
  
  server: {
    port: 3334,
  },
})
```

## Running Locally

### Prerequisites

- Node.js and npm installed
- Access to the Sanity project

### Development Server

1. Navigate to the studio directory:
   ```bash
   cd delpoble-studio
   ```

2. Install dependencies:
   ```bash
   npm install
   ```

3. Start the development server:
   ```bash
   npm run dev
   ```

4. Open your browser to:
   ```
   http://localhost:3334
   ```

### Available Scripts

<ParamField path="dev" type="command">
  Start development server with hot reloading
  ```bash
  npm run dev
  ```
</ParamField>

<ParamField path="start" type="command">
  Start production server
  ```bash
  npm start
  ```
</ParamField>

<ParamField path="build" type="command">

  ```bash
  npm run build
  ```
</ParamField>

## Custom Structure

The Studio uses a custom structure to organize content logically:

```typescript
structure: (S) =>
  S.list()
    .title('Contenido')
    .items([
      // Configuration
      S.documentListItem()
        .title('⚙️ Configuración General')
        .schemaType('siteSettings')
        .id('siteSettings'),
      
      S.documentListItem()
        .title('📄 Footer')
        .schemaType('footer')
        .id('footer'),
      
      S.divider(),
      
      // Pages
      S.listItem()
        .title('🏠 Front Page')
        .child(S.document().schemaType('frontPage').documentId('frontPage')),
      
      // Carta section
      S.listItem()
        .title('📋 Carta')
        .child(S.list().title('Carta').items([...])),
      
      // More sections...
    ])
```

This structure provides:
- Emoji icons for visual navigation
- Logical grouping of related content
- Singleton documents for unique pages
- List views for document collections

## Plugins

### Structure Tool

Provides the custom content structure and document management interface.

### Vision Tool

Enables GROQ query testing directly in the Studio:

1. Click "Vision" in the Studio navigation
2. Write GROQ queries to test data fetching
3. See real-time results from your dataset

Example query:
```groq
*[_type == "frontPage"][0] {
  marqueeText,
  orderLink
}
```

## Environment Variables

CLI configuration is set in `sanity.cli.ts`:

```typescript
export default defineCliConfig({
  api: {
    projectId: 'v8qkjh2b',
    dataset: 'production'
  },
  deployment: {
    autoUpdates: true,
  },
  studioHost: 'delpoble'
})
```
